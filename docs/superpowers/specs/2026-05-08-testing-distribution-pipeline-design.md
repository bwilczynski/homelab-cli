# Testing & Distribution Pipeline Design

**Date:** 2026-05-08
**Status:** Approved

## Overview

Add a CI/CD pipeline that runs tests on every push to `main` and publishes a release to GitHub, with the binary installable via a personal Homebrew tap.

## Goals

- Run `go vet` and `go test ./...` automatically on every push to `main`
- Publish a GitHub Release on every push to `main` with macOS binaries
- Make `hlctl` installable via `brew install bwilczynski/tap/hlctl`

## Non-Goals

- Linux or Windows builds
- Manual release triggers (every push to `main` is a release)
- Semver versioning (CalVer + SHA is sufficient for personal use)

## Architecture

Three components:

1. **GitHub Actions workflow** — orchestrates lint, test, and release
2. **GoReleaser config** — builds binaries, creates GitHub Release, pushes Homebrew formula
3. **Homebrew formula** — auto-generated by GoReleaser, pushed to `bwilczynski/homebrew-tap`

## Versioning

Every push to `main` creates a git tag in the format:

```
v{YYYYMMDD}.{4-char-SHA}
```

Example: `v20260508.774a`

The CI job computes and pushes this tag before invoking GoReleaser. The version is embedded into the binary at build time via ldflags and exposed through `hlctl --version`.

## GitHub Actions Workflow

**File:** `.github/workflows/release.yml`
**Trigger:** push to `main`

Jobs run in sequence:

```
lint → test → release
```

- **lint**: `go vet ./...`
- **test**: `go test ./...`
- **release**:
  1. Compute tag: `v$(date +%Y%m%d).$(git rev-parse --short=4 HEAD)`
  2. Create and push the git tag
  3. Run GoReleaser

**Secrets required:**

| Secret | Purpose |
|--------|---------|
| `GITHUB_TOKEN` | Built-in; creates GitHub Release and uploads assets |
| `TAP_GITHUB_TOKEN` | PAT with write access to `bwilczynski/homebrew-tap`; pushes formula |

## GoReleaser Config

**File:** `.goreleaser.yaml`

- **Builds:** `darwin/arm64` and `darwin/amd64`
- **ldflags:** embed version string into `main.go` via `-X main.version={{.Version}}`
- **Archives:** `.tar.gz` per platform, named `hlctl_{version}_{os}_{arch}.tar.gz`
- **Checksum:** `sha256` checksum file included in release
- **Homebrew brews:** push `Formula/hlctl.rb` to `bwilczynski/homebrew-tap` on branch `master`
- **Formula test block:** `system "#{bin}/hlctl", "--version"`

## Homebrew Formula

GoReleaser auto-generates `Formula/hlctl.rb` in `bwilczynski/homebrew-tap`. The formula:

- Downloads the appropriate `.tar.gz` for the platform
- Verifies the SHA256 checksum
- Installs the `hlctl` binary to `bin/`
- Runs `hlctl --version` as the smoke test

Install command (one-time tap setup already done):

```sh
brew tap bwilczynski/tap
brew install hlctl
```

Upgrade:

```sh
brew upgrade hlctl
```

## Binary Version Flag

The `--version` flag is added to the root Cobra command in `internal/cli/root.go`. The version value is a package-level variable in `cmd/hlctl/main.go`, set to `"dev"` by default and overridden at build time via ldflags.

## Makefile

Add a `test` target for local use:

```makefile
test: ## Run tests
    go test ./...
```
